# Internationalization (Multi-language + Multi-currency)

> Goal: sell worldwide. Both translated content and localized pricing from launch.

## Languages
- Locale-prefixed routes: `/en/...`, `/fr/...`, `/es/...`, `/hi/...` (final list TBD).
- Library: **next-intl** for routing + UI string translation (or Payload localization for
  content fields). Decision finalized during Phase 3 implementation.
- Translatable content (product titles/descriptions, category copy) stored per-locale in Payload.
- `hreflang` alternates emitted for every localized page (ties into SEO.md).

## Currency
- Prices stored in a **base currency** (recommend USD) in the DB.
- Display converted to the visitor's currency (USD, EUR, GBP, INR, ...) using FX rates.
- FX source TBD (e.g. daily-cached rates from an FX API).
- Format per-locale (symbol, decimal/grouping) via `Intl.NumberFormat`.
- Currency selection: auto-detect (geo/headers) + manual switcher; persisted in a cookie.

## Payments × currency
- **Stripe**: charge in the customer's currency where supported (international).
- **Razorpay**: charge in INR (India).
- Routing logic decided in PAYMENTS.md.
