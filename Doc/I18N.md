# Internationalization (Multi-language + Multi-currency)

> Goal: sell worldwide. Both translated content and localized pricing from launch.

## Languages
- Locale-prefixed routes: `/en/...`, `/fr/...`, `/es/...`, `/hi/...` (final list TBD).
- Library: **next-intl** for routing + UI string translation (or Payload localization for
  content fields). Decision finalized during Phase 3 implementation.
- Translatable content (product titles/descriptions, category copy) stored per-locale in Payload.
- `hreflang` alternates emitted for every localized page (ties into SEO.md).

## Currency — IMPLEMENTED ✅
- Base currency is **INR** — admin enters rupee prices (`Products.price`).
- Storefront shows each visitor's local currency, converted from INR (`lib/format.ts`:
  `FX_FROM_INR`, `convertFromInr`, `formatMoney`, `currencyForCountry`).
- **Geo detection:** `proxy.ts` reads Vercel's `x-vercel-ip-country` and stores the chosen
  currency in a `pe_currency` cookie; `CurrencyProvider` + `<Price>` format client-side;
  `CurrencySwitcher` in the header lets visitors override.
- FX rates are **static** for now — wire a live FX feed later.

## Payments × currency
- **Razorpay charges in INR** regardless of display currency. Checkout shows the local currency
  with a small "charged in INR" note. (Stripe dropped.) See PAYMENTS.md.
