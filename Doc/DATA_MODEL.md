# Data Model (Payload Collections)

> The schema lives in Payload collection configs under `src/collections/`. This file is the
> human-readable mirror — update it whenever a collection or field changes.

## Collections (implemented)

### Products
- `title` (localized), `slug` (unique, read-only — derived from title), `sku` (unique Product ID,
  auto `PE-00001`, editable)
- `description` (localized, **textarea** — was rich text; AI can fill it)
- `colors` (array, required) — **unified photos model**: each color row = `name`, `available`
  checkbox, and a nested `images` array (required) of `{ image → Media, default }`. The photo
  marked **default** shows first; selecting a colour on the product page swaps the gallery.
  **There is no separate top-level images array** — all photos live under a colour.
- `price` (number, **INR**), `compareAtPrice`, `category` (relation → Categories, required)
- `collections` (relation → Collections, hasMany, optional) — themed groupings (Diwali, Holi…)
- `sizes` (array: label/stock/sku), `stock`, attributes `fabric`/`color`/`occasions`/`pattern`/`careInstructions`
- `status` (active/oos/archived), `featured`, `badge`, `accentColor` (gradient fallback)
- `seo` group (metaTitle/metaDescription/ogImage); `aiAssistant` UI field (OpenAI generator —
  writes the listing AND auto-generates alt text for every photo)

### Categories
- `title` (localized), `slug`, `image`, `accentColor`, `featured`, `order` (**display order** —
  lower shows first; ties fall back to creation order), `seo`
- `aiSeo` UI field — one-click AI fills `seo.metaTitle` + `seo.metaDescription` from the title
  (hidden from shoppers, used for search). **No visible `description` field** (removed).
- `beforeDelete` hook blocks deletion with a clear message while products still reference the
  category (move/delete those products first).

### Collections
- Themed, cross-category groupings (e.g. "Diwali Edit", "Holi"). A product can belong to many.
- `title` (localized), `slug`, `description` (localized), `image`, `featured`, `order`, `seo`.
- Storefront: `/collections` (index) and `/collections/[slug]` (landing, reuses ProductListing).

### Media
- Uploads stored in **Supabase Storage** (public `media` bucket) via custom adapter.
- `alt` (localized) — **auto-generated by AI** on upload (optional, editable).
- On upload the file is also **auto-renamed** to a descriptive, SEO-friendly slug derived from
  the AI alt text (one vision call covers both alt + filename).

### Orders
- `orderNumber` (PE-xxxx), `email`, `customer` (relation → Customers)
- `items` (product, title/size/qty/unitPrice snapshot), `currency` (**INR**), `subtotal`,
  `shipping`, `tax`, `total`
- `paymentProvider` (razorpay/cod), `paymentStatus`, `providerRef`, `shippingAddress`,
  `fulfillmentStatus`

### Customers (auth)
- Storefront accounts: email, name, phone, `addresses`, `wishlist` (rel → products),
  `cart` (array, account-synced). Email OTP verification: `isVerified` + hidden
  `otpHash`/`otpExpiresAt`/`otpAttempts`; `beforeLogin` blocks unverified.

### Users (auth)
- Admin/staff accounts for the Payload admin panel.

### Messages
- Contact-form submissions: name, email, phone, subject, message, status. Public create; admin read.

### Settings (global)
- `heroImages` (array → Media, homepage slideshow), store name, contact/WhatsApp, socials,
  commerce defaults (base currency, tax). Country shipping lives in `lib/shipping.ts`.

## Conventions
- **Base currency is INR**; storefront converts to the visitor's currency at render time.
- Localized fields use Payload localization for translated content.
- Slugs are unique and URL-safe; used directly in storefront routes.

> STATUS: **Implemented** (2026-06-20). Collections live in `src/collections/*` and the
> `Settings` global in `src/globals/Settings.ts`. Differences from the original plan:
> - Products use a `status` field (active / oos / archived) instead of Payload drafts
>   (drafts' `_status` enum collided with our `status` enum on Postgres).
> - Products carry an `accentColor` gradient fallback used until real images are uploaded.
> - `slug`, `seo` are reusable fields (`src/fields/`). Access control in `src/access/`.
> - Storefront reads go through `src/lib/queries.ts` (mapped to decoupled `StoreProduct`/
>   `StoreCategory` shapes), not the generated payload types.
